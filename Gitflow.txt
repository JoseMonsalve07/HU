Gitflow

Crear rama:
1. pararse en develop
git checkout develop

2. crear la rama con el nombre necesario
git checkout -b feature/setup-inicial-task1


🟢 1. Verifica que estás en la rama correcta
git status


🟡 2. Agrega tus cambios al stage
Cuando ya tengas los archivos creados/modificados:
git add .


🟠 3. Crea un commit descriptivo
Usa mensajes claros con verbo en presente:
git commit -m "Inicializa proyecto Node.js + TypeScript y configura tsconfig.json"


🔵 4. Sube la rama al remoto (solo la primera vez)
Ya la creaste localmente, así que ahora ejecútalo una vez:
git push -u origin feature/setup-inicial-task1

A partir de aquí, para los siguientes pushes puedes usar simplemente:
git push


🟣 5. Crea el Pull Request (PR) en GitHub
Ve a tu repositorio en GitHub.
Verás un botón amarillo o verde que dice “Compare & pull request” → haz clic.
Asegúrate de que diga:
base: develop
compare: feature/setup-inicial-task1

Escribe un título claro:
HU1 - Setup inicial del proyecto (Task 1 completada)

En la descripción puedes poner un resumen como:
✔️ Se inicializó el proyecto con Node.js + TypeScript.
✔️ Se configuró tsconfig.json.
✔️ Se instalaron dependencias base: express, ts-node, nodemon.

Envía el PR.


🔴 6. Revisión y merge
Cuando se apruebe el PR (o si estás trabajando solo), haces el merge en GitHub hacia develop.
Luego, vuelve a tu entorno local y actualiza tu rama develop:

git checkout develop
git pull origin develop

Esto garantiza que tu entorno local está sincronizado con lo que hay en GitHub.


⚪ 7. Limpieza opcional
Una vez fusionado el PR, puedes borrar la rama feature:

git branch -d feature/setup-inicial-task1        # borra localmente
git push origin --delete feature/setup-inicial-task1   # borra en remoto
